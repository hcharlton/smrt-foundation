# Overview
This experiment mirrors ssl_58 with a 512 context window autoencoder experiment. The purpose is to generate model weights from mixing a Sequell II sample (DA1), and a newer Revio sample (Yoran). This will allow us later to determine if there is better generalizability from mixing samples, or whether it's enough to just pretrain on one sample. 
## Caveats
### Normalization
Currently (worth doublechecking) the normalization is calculated by taking a sample of the dataset and then z-normalizing the rest of the dataset online based on that sample. For a two sample dataset, should we take a sample of the combined dataset and then use those stats for normalizing? Or something more complicated. Regardless, we should generate a comparison of stats beforehand. I am not sure what the the best practice is for handling something like this. However it's handled, it will have implications for a downstream task since we want to keep the normalization strategy used in pretrained applicable there. The downstream dataset currently comes from Sequell II machines, but the whole point is we want future flexibility, so we want to account for using the encoder for a Revio task in the future.
